# pmo-status-report-template

Professional PMO Weekly Status Report template built for government ICT project delivery in Malaysia. Designed to reflect real reporting standards used by Project Coordinators and PMO teams across government and enterprise ICT programmes.

All sample data is fictional and constructed for portfolio purposes.

---

## Overview

| Field | Details |
|-------|---------|
| Template Type | PMO Weekly Status Report |
| Format | Microsoft Excel (.xlsx) |
| Sheets | 2 (Weekly Status Report, Instructions) |
| Target Roles | Project Coordinator, PMO Analyst, Project Executive |
| Context | Government ICT project delivery, Malaysia |

---

## What Is Included

### Sheet 1 — Weekly Status Report

| Section | Description |
|---------|-------------|
| Document Control | Document ID, version, status, classification, prepared by and date |
| Distribution List | Name, role, organisation, email, copy type (To / CC), date distributed |
| Project Information | Project name, PM, client/agency, phase, reporting period, week number |
| Overall Project Status | RAG status across 6 areas with summary comments |
| Milestone Status | Baseline, forecast, and actual dates with status and variance remarks |
| Activities Update | Completed this week and planned next week side by side |
| Risks and Issues Log | ID, type, description, RAG, owner, target date |
| Budget Status | RAG per area, approved budget, spent to date, auto-calculated remaining and utilisation |
| Decisions and Approvals | Formal log of decisions and approvals required from leadership |
| Revision History | Version control log for document changes |
| Sign-Off Block | Prepared by, reviewed by, approved by with signature and date fields |

### Sheet 2 — Instructions

Step-by-step guidance on how to complete each section correctly. Includes RAG legend with definitions for GREEN, AMBER, RED, and CLOSED statuses.

---

## RAG Status Reference

| Status | Meaning |
|--------|---------|
| GREEN | On track. No issues. Proceeding as planned. |
| AMBER | At risk. Action required. Escalation may be needed. |
| RED | Off track. Immediate escalation and corrective action required. |
| CLOSED | Risk or issue resolved. Retained for audit trail. |

---

## Milestone Status Reference

| Status | Meaning |
|--------|---------|
| COMPLETE | Milestone achieved. Actual date recorded. |
| IN PROGRESS | Milestone currently being worked on. |
| NOT STARTED | Milestone not yet reached. |
| DELAYED | Milestone has slipped from baseline date. |

## Document Control Fields

| Field | Description |
|-------|-------------|
| Document ID | Unique identifier e.g. PMO-SR-001. Increment per project. |
| Version | 1.0 = initial. 1.1 = minor update. 2.0 = major revision. |
| Status | Draft = under review. Final = approved for distribution. |
| Classification | Confidential for most government ICT projects. |

## How This Reflects Real PMO Practice

- RAG reporting per status area is standard across Malaysian government ICT programmes including projects under MAMPU, MDEC, and federal agency oversight
- Milestone tracking with baseline vs forecast vs actual reflects contractual reporting requirements under LOA-based government ICT contracts
- BRS, SRS, UAT, FAT, Go-Live milestone sequence follows standard Waterfall SDLC gate structure used by Malaysian system integrators
- Distribution list and revision history are standard document control requirements for ISO-aligned organisations and government vendors
- Sign-off chain with prepared, reviewed, and approved roles reflects formal governance requirements on government project deliveries
- Budget status uses RAG with figures rather than figures alone, matching how budget health is communicated in steering committee reporting
